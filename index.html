The Deets

What is it?

The Deets is an online get-together organizer. The Deets keeps the details of an event in one place; like the date, place, guest attendance, theme, and other specific details.

Wondering what's being brought to the potluck? Not quite sure who's going to bring decorations? Too much noise and fodder on other alternative event organizer websites? The Deets helps you keep event details simple, and easy to find.

How does it work?

In order to view, or be invited to any events, a user must create an account or be logged in. Every user has a home page where they can see the events for that month, notifications, and also able to search for events created by friends.

The Deets protects events from strangers. So to be able to attend or find an event, you must be friends with the owner. The Deets further protects events, even from friends, by having to know a specific search key in order to find the event.

At the event page users can see the details of said event such as: address, theme, description, attending guests, comments, and categories needed for the event. The Deets helps every event be organized, and enjoyable for everyone involved.

I created The Deets to organize my friends and I annual "Friendsgiving". So, attending guests can sign up to bring items to an event for a certain category made by the owner.

Personal thoughts

The Deets is a passion project that I look forward to work on. This iteration of the site in particular, is something I really wanted to complete. This web app showcases the more up to date skillset I have compared to some of the other projects I have on display on my portfolio. Also, it's completely workable as is. However, I would like to add a bit more features.

The Deets is great for personal use. The more I code for front-end development, the more I get a feel of how I want certain things to be organized. Such as OOP based JS, or how to code the SASS. I hope that you the reader will also find The Deets useful, amongst others that I will tell about this website.

JK Chef Collection

What is it?

JK Chef Collection is an online web application for chefs, cooks, bartenders, and waiters to apply to JK Chef Collection. I was contacted by Jack Kelly through a colleague to build a site to handle the applications, information, and organization of their culinary clients.

How does it work?

After a potential client fills out the application, the application is "pending". One of the approved accounts to review applications then decides if the application will be accepted or denied.

JK Chef Collection employees can search for chefs, cooks, bartenders, kitchen help, or waiters/waitresses by checking off boxes. The search can further be refined by searching for: salary range, types of cuisine, languages known, and availability.

Once JK employees find clients that match their desired criteria. They can use the information provided from the application to contact them and work out a contract.

Personal thoughts

JK Chef Collection seemed like a huge challenge when I first heard what was being requested. However, the more I spoke to Jack and Jeff, the more details were worked out. Before we agreed on terms for the website, I made wireframes based on what they were requesting, then asked for feedback. This went on a few times until it represented what they were looking for. Then, we agreed to go on and build the full site.

Communication played a huge role at keeping this project run smooth. Any ambiguities I made sure to double check if that was what they wanted. However, at the same time, I tried my best to make good judgement calls at improving certain aspects that they wanted to achieve. They always appreciated the extra effort to make it easier for them, and I very rarely had to double back to make any major changes.

I hope that other contracts run as smooth as this one did.

Sandwich Builds

What is it?

Sandwich builds is for the sandwich craftsman. It's a node web application that helps you create and rate sandwich recipes upon sign up.

How does it work?

After a user signs up or logs in, they can view passed recipes by clicking on their names. When you click a sandwich name, the recipe appears in an overview where you can view it.

If a user wants to create another recipe, they simply have to scroll down and start plugging in information for the sandwich. There is a navigation bar where the user can click on multiple "segments" for the recipe. Materials, stats, and directions to name a few. Before the user saves the recipe they view an overview of what the sandwich is.

A user would also be able to search for sandwich recipes based on stats, or materials that they already have. I believe there is a lot potential for this site to be frequented by the sandwich connoisseur. There are a lot of additional features I have in mind to add. But there are other things that I want to improve first. With that said, I will just leave this project as inactive for now.

Assassins The Game

What is it?

Assassins is a live action role playing game where a group of people receive a "target" who they must eliminate. Upon doing so, you receive their target and continue. The rules vary from game to game depending on the moderator, however there are a base of rules that is widely used.

This web application organizes a game for both players and moderators.

How does it work?

Moderators are able to create a game with a passcode and specific rules if they choose. They can also see how many players have joined the game, their information, and communicate with the game. When the moderator feels that the matchup is fine, they can start the game.

Assassins can view other assassins still in the game, or eliminated, and post comments under their guise for fun. When assassins are still in the game, a fake picture is there for everyone to see. But once they are eliminated their real photo is revealed.

Assassins is based on scouts honor. So if the assailant has eliminated their target, they press a button to confirm the kill. The victim then clicks a final confirmation that they have indeed been eliminated. Once both the assailant and survivor agrees on the kill, the next target is assigned.

Personal thoughts

Assassins is a fun project that I always wanted to code and complete. The logic can definitely be cleaned up for users joining as assassins. And also how the targets are assigned. I do plan on adding features such as: improved UI, clearer comments, and private messaging.

Throw Words

What is it?

Throw Words is fast paced team charades. Try and get as many points as you can before your team timer runs out!

How does it work?

First, you put in at least 3 words per person that is participating in the game into the array of words to act out. Then, once you feel like there are enough words, you press start.

Every turn a team will have a guesser. If your guesser says the Throw Word then your team gets the point. It will then be the other teams turn.

The guesser will now quickly rotate with another member of their team, and will be they will be the guesser of their next turn. The guesser can opt to pass in order pause their team's timer. However, the other team has the chance to get the point. In the event both team's guessers pass consecutively, neither team will get the point and the next Throw Word will appear on screen.

The team with the most points when both teams timer's expires, wins!

Personal thoughts

Throw Words started off as a side project to get used to crudding simple information. However, I found it to be fun to gamify it somehow. Eventually I ended up with Throw Words!

I am fairly proud of the outcome of this, and I will definitely find ways to improve the gameplay. Maybe different hinderences on acting it out? Such as no talking, or one word only for that turn.

Ulimately, I see Throw Words being a fun fast paced charade alternative with different rules and quirks per turn.

Coding Bootcamp Reviews

What is it?

View, find, and review the best coding bootcamp for you!

Coding Bootcamp reviews is a website meant to help people that are looking to make the leap to a career in code. A user can view coding bootcamps in the Bay Area and see reviews made by other students who have taken to the program. A "Rate My Professor" for coding bootcamps if you will.

How does it work?

A user can search for a coding bootcamp and view an overall rating, general information, and a percentage of ratings for a few specific rating topics. These topics include if the bootcamp was worth the price, location score, job search help, and campus and facilities.

A user can view reviews in detail by scrolling down the page. Here you can find reviews left by students that has attended the bootcamp with personal ratings, and a personal opinion if they choose to leave one.

If a user wants to leave a review of a bootcamp, they would have to create an account and write a review. The review would then go towards the ratings of the bootcamp and the rating would adjust.

Personal thoughts

This project was definitely interesting. It was a group project that had it's ups and downs.

Let's start with the downs. Usually a common problem is too many cooks in the kitchen. However, in this instance no one was really calling the shots. And if someone were to start to move towards making decisions, that decision would soon be undone by the group. Another problem was that not everyone was up to par in merging git branches without causing conflicts. You mix that with a few people stepping on eachother's toes and taking goals that weren't theres, caused a bit of a stressful situation.

Alright. The ups. First of all, the downs were resolved by openly communicating. Also, by being extremely clear on who was doing which task. Pair programming helped communication, and lesen errors as well. When the group finally started to click and get morning scrums rolling, it was really nice to have things be completed without having to worry about it. It was like coming to an in progress puzzle with your piece to find that other pieces have been filled out while you were finding your piece. The satisfaction of working with others and accomplishing something is definitely more enjoyable for me than working solo.

SF Quake Search

What is it?

Search through an earthquake API for recent earthquakes. The API only catalogues earthquakes within the last month, unless you pay for the full API.

How does it work?

Input the date of that you want earthquakes to appear since. Followed by the minimum magnitude of the earthquake. Then finally by X degrees outward, since the focal point of the map is in SF.

Users can click and drag around the map to see markers of where the earthquakes are. Or they can click on the results on the right hand side to be taken to that specific earthquake marker.

Personal thoughts

This project was to practice handling API and json data. Then taking that data and using it to make something cohesive. I used the earthquake.usgs.gov API with the Google maps API.

This project really helped me understand the basics of handling an API and using callbacks to handle asynchronous functions.

Event Wall

What is it?

Event wall is a media wall to keep your guests engaged throughout the event. Guests can tweet using a specific hashtag, or visit the stream url to send a message that will appear on the wall. Some ideal events it can be used for are wedding receptions, Sweet 16s, Cotillions, work events, and conventions.

Guests can read funny updates, or see the event from other guests point of view.

How does it work?

Users that sign up to EventWall can create a stream to display during there event. They choose the hashtag that will be tracked on Twitter that will be used for the event. Guests can also go to a specific url to submit messages and photos specifically to the website. A banner, and logo can also be uploaded to display on stream.

The messages are looped through on stream that can be displayed on a wall. There are controls that a user can use to go back in tweets, go forward, pause the stream, or even remove certain tweets/messages.

Personal thoughts

This project was originally for my friends Juan and Lisa for their wedding reception. In fact, the project folder locally is still called "LisaFoundTheJuan". It was brought to my intention that there was a projector, but they weren't exactly sure what to use it for. So, I thought having a stream of tweets, pictures, and messages from all of the guests through out the day was a fun way to use the projector.

I thought it would be really cool to have all of these guests have this memorable event in their heads, but then they get to see it from a different perspective. Sure they saw this, but did they see it from over there. Maybe they didn't even know of something that happened and now they get to see it.

The more I coded, I thought it would be great if I can make it a full blown full stack web application as opposed to a personal stream. Where users can input their own logos, banners, and hashtags. I got the basic logic of it down, I just have to improve the front end.

Coin Race

What is it?

Race against your friend to collect coins to impress the princess!

How does it work?

The players move around to collect coins that appear in the same region for both players. Whoever gets to the coin first gets the coin as it moves to another region on the grid.

Each player enters their name, and chooses a character. Once they are ready they click "lock in". Each player has a set of controls that they can use to move up, down, left, or right. Once they feel comfortable with the controls they can start the game.

On the grid are also spooky monsters that if you run into them, you are sent to the beginning of the grid. Avoid them, and collect enough coins to unlock the princess. Run to her with your hard earned coins to win the game.

Personal thoughts

This was my first project that I completed in code. The whole process, including its first iteration took two weeks. Square Racer 1 was very very simple. And just featured a square moving across a grid with one button. A mashing game against the other player.

Over the weekend of that first week I thought it would be cool to have objects to avoid so I thought of a process to keep track of the square in a multi row grid. Then, I found a learned about setInterval and set logic to place boxes to occupy certain areas.Which led to Square Racer 2.

Eventually I thought of a task for these two squares to complete before the can simply make it to the finish line. I added the coin feature to make it really fun over the course of week 2. I added some CSS and ended up with CoinRacer.

I was really excited adding features and improving this project. It was the first thing I ever built with code and it was truly truly rewarding. I think of this often when I think of how far I've come since the beginning, and also how hungry I was to learn more in the beginning.
